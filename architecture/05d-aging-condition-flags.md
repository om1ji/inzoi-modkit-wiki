# AgingConditionFlags — маска возрастов

`AgingConditionFlags` — поле `InteractionBag` (тип `int32`), которое задаёт, **каким возрастным
этапам доступно действие**. Это не число-«уровень», а **битовая маска**: каждый бит отвечает за один
этап взросления, и включённый бит означает «этому возрасту — можно». Значения вроде `508` или `504`
выглядят загадочно, пока не разложить их на биты.

За набором этапов стоит enum `EB1GrowthStep` — девять стадий жизни зоя по порядку.

## Карта битов

| Бит | Значение | `EB1GrowthStep` | Возраст |
|---|---|---|---|
| 0 | `1` | `Infant` | младенец |
| 1 | `2` | `Toddler` | ясельный |
| 2 | `4` | `Child` | ребёнок |
| 3 | `8` | `LowTeen` | младший подросток |
| 4 | `16` | `HighTeen` | старший подросток |
| 5 | `32` | `Youth` | юность (молодой взрослый) |
| 6 | `64` | `Adult` | взрослый |
| 7 | `128` | `Old` | пожилой |
| 8 | `256` | `Elderly` | старик |

Итоговое значение — это **сумма** значений всех разрешённых этапов. «Разрешить всем» = `1+2+…+256 =
511`.

## Как читать значение

Разложи число на слагаемые-степени двойки — получишь список разрешённых возрастов.

**508** = `4 + 8 + 16 + 32 + 64 + 128 + 256` = `Child, LowTeen, HighTeen, Youth, Adult, Old, Elderly`.
То есть «все, кроме младенца и ясельного». Логично: `Toilet01_Use`, `Sleep_Bed`, `Computer_Use` — то,
что доступно каждому, кто уже ходит и пользуется мебелью.

**504** = `508 − 4` = то же самое, но **без `Child`** → `LowTeen` и старше. Так помечены «взрослые»
действия: `Use_Mic_Sing`, `Coop_Pillow_Fight`, `Self_Busking_Song`, тест на беременность.

**4** = только `Child`. Ровно это стоит у `C_Hide_and_seek` — детская игра в прятки, недоступная
никому, кроме детей.

## Реальные значения из игры

| Значение | Разрешено | Примеры |
|---|---|---|
| `511` | вообще все | (полностью открытые действия) |
| `508` | `Child`…`Elderly` | `Sleep_Bed`, `Computer_Use`, `Toilet01_Use` |
| `504` | `LowTeen`…`Elderly` | `Use_Mic_Sing`, `Self_Squat`, `Self_Busking_Song` |
| `4` | только `Child` | `C_Hide_and_seek` |

```{note}
Официальный enum упоминает и «сборные» группы-диапазоны (`Baby`, `OverAdult`, `OverOld` и т.п.) —
именованные маски сразу на несколько этапов для удобства. Но в самих строках хранится обычное целое
число, поэтому модеру достаточно оперировать суммой битов из таблицы выше.
```

## Как собрать своё значение

Выбери разрешённые этапы и **сложи** их значения из колонки «Значение»:

- только взрослые (`Adult`) → `64`;
- подростки и старше (`LowTeen`…`Elderly`) → `8+16+32+64+128+256 = 504`;
- ребёнок и старше (`Child`…`Elderly`) → `4+…+256 = 508`;
- совсем все → `511`.

```{tip}
`AgingConditionFlags` — **скаляр (int)**, а не массив. Это один из немногих «интеракционных» полей,
которые MCP `set_data` пишет надёжно, без проблемы стрингификации массивов (см. {doc}`13-gotchas`).
Так что возрастной гейт своего действия можно менять прямо через MCP.
```

## Чем отличается от условий

`AgingConditionFlags` и `InteractionVisibleConditionIds` делают похожее (ограничивают доступ), но это
разные механизмы:

- **`AgingConditionFlags`** — быстрый встроенный фильтр именно по возрасту, зашитый в одно число на
  самой строке.
- **Условия `Condition_*`** — универсальный движок для всего остального (эмоции, теги, баффы, роль,
  локация); см. {doc}`05b-conditions`.

Проще говоря: возраст решается маской, а всё сложное — условиями. Для «только взрослым» достаточно
маски, отдельное условие заводить не нужно.
