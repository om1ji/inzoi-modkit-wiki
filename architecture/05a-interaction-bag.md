# Что такое InteractionBag

`InteractionBag` — это **один пункт меню** предмета и одновременно самый «богатый» слой всей цепочки
интеракций. Если `ObjectSelectionSet` просто перечисляет, какие пункты вообще доступны, а
`Interaction`/`Action` отвечают за механику выполнения, то в `InteractionBag` собрана почти вся
«личность» действия: как оно называется, что даёт персонажу, при каких условиях показывается, как
ветвится и какие эффекты запускает.

Название («bag», мешок) отражает суть: это не одно действие, а **набор** — обёртка, которая может
разворачиваться в несколько веток с вероятностями и цепочками продолжений.

```
ObjectSelectionSet (список пунктов) 
  → InteractionBag (сам пункт: награда, условия, ветвление, скрипты)
      → Interaction (как выполнять) → Action (анимация)
```

Где он лежит: у сложных предметов — в именной таблице (`InteractionBag_Computer`), у простой мебели —
в **плоской** `InteractionBag` без суффикса. Если `get_row` не находит строку в именной таблице,
проверяй плоскую (см. {doc}`05-interaction-pipeline`).

## Что внутри: поля по смыслу

Значения ниже — реальные, из двух живых примеров: простого `Toilet01_Use` и насыщенного
`Computer_Use`.

### Название и вид в UI

- **`DisplayInfo`** — блок отображения. `DisplayTextId` и `DisplayCategoryTextId` — ссылки в систему
  локализации, откуда берётся видимый игроку текст. Здесь же иконки (`Icon`, `PanelIcon`), маркеры и
  флаг `bHideUI` (скрыть пункт из меню, оставив логику).
- **`ActionHintTextForSLM`** — человекочитаемая подсказка для ИИ-автономии («Using the toilet»,
  «Playing a game on the computer»): по ней система решает, когда NPC сам выберет это действие.

### Награды — что получает персонаж

Здесь и видно, зачем действие вообще нужно:

- **`InteractionRewards`** — восстановление мотивов. У унитаза одна награда (`Toilet +15`), у
  компьютера сразу три (`Fun +10`, плюс рабочие метрики `Customer_Service_Usage`,
  `Staff_CustomerService`).
- **`PreferenceRewards`** — тонкая настройка «под характер»: бонусы к чертам (`TraitRewards`:
  `Independent +20`, `Entertainer +20`), к предпочтениям (`Gaming`: кто любит — `+8`, кто не любит —
  `−5`) и к эмоциям (`EmotionRewards`: поднять `Excited`, погасить `Bored`). У унитаза здесь скромно —
  небольшой шанс дискомфорта.

### Условия — когда пункт вообще доступен

Три независимых «ворот», каждое — список ID строк `Condition_*`:

- **`InteractionVisibleConditionIds`** — видно ли пункт в меню.
- **`InteractionEnableConditionIds`** — активен ли он (можно кликнуть).
- **`EnqueueConditionIds`** — можно ли поставить в очередь.

У `Toilet01_Use` все три пустые — поэтому он доступен почти всегда. У `Computer_Use` в них висит по
четыре условия (например `DC_PC_Use_Condition`, `Check_NotClubDutyAtSchool`) — компьютер доступен не
в любой ситуации. Пустой список = «нет ограничений».

```{note}
`AgingConditionFlags` (у обоих `508`) — отдельный механизм: битовая маска допустимых возрастных
групп, зашитая прямо в число, а не ссылка на `Condition`.
```

### Ветвление и цепочки — `InteractionSequence`

Сердце «мешка». Это массив веток, у каждой — своя вероятность (`Probability`) и опциональное условие
(`EnableConditionId`). Игра бросает «кубик» и выбирает ветку:

У `Toilet01_Use` три ветки:
- `Breakable_Water` (0.499) → обычное использование
- `Toilet_Phone` (0.5) → использование с телефоном
- `Break` (0.001, только дома) → поломка

Внутри ветки поле `InteractionInfos[].InteractionId` указывает, какую `Interaction` запустить (слой
ниже). А `NextInteractionBagId` связывает пункты в **цепочку**: обе обычные ветки унитаза ведут в
`Next_WashHand04_AfterUse_Toilet` — то есть после туалета персонаж автоматически идёт мыть руки. Так
из отдельных пунктов собираются длинные сценарии поведения.

### Скрипты — эффекты действия

- **`StartScriptIds`** — срабатывают в начале действия.
- **`FinishScriptIds`** — по завершении. Это **точка подключения №1** для эффектов (награды деньгами
  и т.п.). Показательно: в живых данных `Toilet01_Use.FinishScriptIds` уже содержит
  `["Object_Finish_Toilet_VeryLow", "AddCurrency_50"]` — второй ID дописан модом «Toilet Money».

```{seealso}
Вторая, независимая точка подключения скриптов — на уровне `Action` (`FinishScriptIdList`). Разница
между ними разобрана в {doc}`../case-studies/interaction-layers`.
```

### Автономия, валюта, флаги

- **`AutonomyGroupTag`** / **`AutonomyCooltimeMin`** — как и как часто NPC выбирает действие сам. У
  компьютера кулдаун `60` минут, у унитаза `0` (без ограничения).
- **`InteractionBagTags`** — теги для систем частоты (`IR_Freq_Normal`/`IR_Freq_Low`), событий,
  эмоций. У компьютера их целых десять — он завязан на множество игровых систем.
- **`ConsumeCurrencyId`** / **`ConsumeCurrencyValue`** — гейт стоимости: сколько нужно денег, чтобы
  *начать* действие. Это **не начисление** — см. предупреждение ниже.
- **`RepeatCount`**, **`bRejectable`**, **`bUniqueInQueue`**, **`bNonCancelable`** — мелкие флаги
  поведения в очереди действий.

```{warning}
`ConsumeCurrencyId`/`ConsumeCurrencyValue` — это проверка «хватает ли денег на старт», а не способ
дать деньги. Чтобы **начислить** валюту, нужен `Command: AddCurrency` в `Script_Interaction`,
подключённый через `FinishScriptIds`. Подробно — {doc}`05-interaction-pipeline`.
```

## Простой против сложного: `Toilet01_Use` vs `Computer_Use`

| Аспект | `Toilet01_Use` | `Computer_Use` |
|---|---|---|
| Награды мотивов | 1 (`Toilet +15`) | 3 (`Fun`, + рабочие метрики) |
| Награды черт/предпочтений | нет | черты, `Gaming`, эмоции |
| Условия (visible/enable) | пусто — доступен почти всегда | по 4 условия |
| Ветки `InteractionSequence` | 3 (обычно / телефон / поломка) | 2 (игра / поломка) |
| Цепочка после | → мытьё рук | нет |
| Кулдаун автономии | `0` | `60` мин |
| Теги | 2 | 10 |

Вывод: у базовой бытовой мебели почти все продвинутые поля стоят в нейтральном состоянии, и работает
только необходимое (награда, ветвление). У «системного» предмета вроде компьютера тот же формат строки
задействован куда плотнее — условия, черты, автономия, привязка к бизнесу. Формат один, «наполненность»
разная.

## Как добавить или изменить InteractionBag в моде

- **Изменить существующий пункт** (награду, добавить эффект) — Override строки: например дописать ID в
  `FinishScriptIds`, как это сделал «Toilet Money».
- **Добавить новый пункт меню** — создать новую строку `InteractionBag` (проще всего дюплицировать
  простую вроде `Toilet01_Use_Flush`), затем прописать её ID в `ObjectSelectionSet` нужного
  состояния. Полный пошаговый разбор — в {doc}`../case-studies/interaction-layers`.

```{caution}
Поля-массивы и структуры `InteractionBag` (`InteractionSequence`, `FinishScriptIds`,
`InteractionRewards`, `InteractionEnableConditionIds`) MCP `set_data` стрингифицирует — правь их
вручную в редакторе ModKit. Скаляры (`RepeatCount`, `AutonomyCooltimeMin`, флаги) через MCP надёжны.
См. {doc}`13-gotchas`.
```
