# Частота автономии: теги IR_Freq_*

`IR_Freq_*` — это теги в `InteractionBag.InteractionBagTags`, которые говорят системе автономного
поведения, **насколько часто зой будет сам, без участия игрока, выбирать это действие** в свободное
время. Это «вес спонтанности»: чем выше ступень, тем охотнее NPC займётся этим сам, когда нечем
заняться. К меню игрока частота отношения не имеет — только к самостоятельному поведению.

Важно: это **строковый тег, а не число**. Отдельной таблицы-реестра с весами нет — соответствие
«тег → вес» зашито нативно в движок. Поэтому набор ступеней ниже собран **эмпирически**, прямым
сканированием реальных `InteractionBag` через MCP.

## Лестница частот (от частой к редкой)

| Тег | Насколько часто | Живые примеры |
|---|---|---|
| `IR_Freq_Normal` | обычная — верхняя ступень | `Toilet01_Use`, `Sleep_Bed`, `Toilet01_Use_Uneasy_Lv3` |
| `IR_Freq_Low` | низкая | `Computer_Use`, `Computer_Use_Bored_Lv1`, `Computer_Use_SandBoxGame` |
| `IR_Freq_VeryLow` | очень низкая | `Use_Mic_Sing`, `Object_RainPuddle01_Playing`, `Self_Rain_Poetry`, `Object_Bed_Disinfect_Normal` |
| `IR_Freq_Rare` | редкая | `Nature_Looking_Park_Fountain`, `Self_TakeSelfie_Perspective`, `SnowMan_Kicking` |
| `IR_Freq_VeryRare` | очень редкая | `Object_Toilet_Cry_Self`, `Object_Toilet_Irritated_Self` |

```{note}
Ступеней `High`/`VeryHigh` в данных **не встретилось** — даже базовые потребности (сон, туалет) стоят
на `IR_Freq_Normal`. Похоже, `Normal` — это потолок, а вся шкала уходит вниз, в сторону всё более
редких «необязательных» действий. Список из пяти ступеней подтверждён на ~35 разных пунктах, но
теоретически может быть неполным (словарь нативный).
```

Закономерность видна по смыслу: обязательные, «жизненные» действия (`Toilet01_Use`, `Sleep_Bed`) —
`Normal`; необязательные развлечения (компьютер, микрофон) — `Low`/`VeryLow`; редкие атмосферные
мелочи (посмотреть на фонтан, попинать снеговика) — `Rare`; и совсем исключительные эмоциональные
реакции (поплакать в туалете) — `VeryRare`.

## Когда тега IR_Freq_* нет вообще

Далеко не у каждого пункта есть частотный тег. В выборке без него оказались, например, `Self_Squat`,
`Self_Warmup`, `Self_Ambition_*`, `Coop_Pillow_Fight`, `AutoFind_Fridge_Pickup_Food`. Это не значит
«никогда» — просто такими действиями управляет **не частотный вес, а другие системы**:

- **`Hope_*` / `HopeLife_*_Want`** — действие всплывает, когда у зоя есть соответствующее желание.
- **`Ambition_*`** — завязано на текущую амбицию/жизненную цель.
- **`Event_*`** — привязано к событию (`Event_Sleeping`, `Event_Busking`, `Event_School_PE`).
- **`Trait_*` / `Preference_*`** — тянется чертой характера или предпочтением (`Preference_Gaming`).
- **`Mission_*`** — часть квеста.
- **потребности/coop/контекст** — сон и еда управляются срочностью мотива, а `Coop_*` и `AutoFind_*` —
  системой поиска цели, а не спонтанным выбором.

То есть `IR_Freq_*` — это вес для «филлерного» поведения от скуки. Действия, у которых есть более
сильный движок (желание, амбиция, событие, потребность), в частотном теге просто не нуждаются.

## Родственные ручки автономии

Частота — лишь одна из настроек самостоятельного поведения. Рядом с ней в `InteractionBag` работают:

- **`AutonomyGroupTag`** (`AGTag_Toilet_Use`, `AGTag_Computer`) — группа однотипных действий для
  автономии.
- **`AutonomyCooltimeMin`** — сколько минут NPC не повторяет действие (у компьютера — `60`, у унитаза
  — `0`).
- **`Lock_Emotion_*`** — теги-блокировки: запрещают действие при определённой эмоции (например
  `Lock_Emotion_Sad_Lv3` — не сядет играть, если сильно грустит). По сути обратная сторона частоты —
  не «как часто», а «когда нельзя».

## Что это значит для мода

- Хочешь, чтобы твоё действие зои **иногда делали сами** — добавь в `InteractionBagTags` подходящую
  ступень: `IR_Freq_Rare` для «изредка», `IR_Freq_Normal` для «регулярно».
- Хочешь, чтобы действие было **только по клику игрока** — не добавляй ни `IR_Freq_*`, ни
  `AutonomyGroupTag`. Тогда автономия его не подхватит.
- Для тонкой настройки скомбинируй с `AutonomyCooltimeMin` (пауза между повторами) и `Lock_Emotion_*`
  (когда не выбирать).

```{caution}
`InteractionBagTags` — это массив, и MCP `set_data` его стрингифицирует (см. {doc}`13-gotchas`).
Добавлять/править теги надёжнее вручную в редакторе ModKit; MCP хорош для чтения и инвентаризации, чем
и собран список выше.
```
